3. The Agentic Pipeline

To gather this data, an autonomous pipeline of specialized AI agents was built using Python. The entire infrastructure costs $0 to run, utilizing free-tier AI and Search APIs.

What the Agent Does:

  • Context Gathering: Google Search via the Serper API simulates a developer searching for API documentation and developer portals.
  • Web Scraping: Jina Reader API visits the top URLs and bypasses JavaScript-heavy SPAs to return clean, LLM-optimized Markdown text.
  • Structured Extraction: Groq, Gemini 2.5, and Sarvam APIs read the documentation and extract structured JSON utilizing zero-shot prompting techniques.
  • Validation Loops: The instructor Python library enforces strict Pydantic JSON schemas, forcing the LLM to retry on validation failures.
  • Macro-Analysis: A dedicated Analyzer Agent processes the entire dataset to perform clustering, identifying the structural patterns presented above.

Where a Human Was Needed:

  • Pipeline Architecture: A human wrote the Python pipeline (using standard async/await patterns) and tuned the Pydantic schemas.
  • Rate Limit Handling: Due to free-tier API quotas on Groq and Gemini, human intervention was required to swap out LLM keys and adjust model endpoints (migrating from Groq to Gemini to Sarvam 105b mid-flight).
  • Deduplication: A human manually noticed minor dataset duplications in the checkpointing logic and wrote a one-off Python script to clean the final JSON output.

4. The Proof (Run it yourself)

The entire pipeline is open-source and runnable locally via a single command line interface. It is robust to failures, saving state incrementally.

$ git clone <repo-url>
$ cd composio
$ uv sync
$ uv run python main.py all

Running this will sequentially spin up the Researcher, Verifier, Analyzer, and Dashboard Generator.

5. Verification & Accuracy Checks

To ensure the data is trustworthy, a dedicated Verification Agent was spawned to sample 20% of the dataset and independently audit the Researcher Agent's claims against the raw documentation URLs.

The Accuracy Jump: Static vs Agentic

Phase 1: Static Verification

Blindly reading top 3 marketing URLs.

  • Primary Auth Accuracy: 25.0%
  • Buildability Accuracy: 31.6%

Phase 2: Agentic ReAct Loop

Iterative web searching & deep-link scraping.

  • Primary Auth Accuracy: 71.4%
  • Buildability Accuracy: 66.7%

How We Improved It: The Agent Loop

The Phase 1 accuracy was unacceptably low due to Context Starvation. The initial script fetched the top 3 Google URLs, which were often high-level pricing pages rather than deep API references, forcing the LLM to hallucinate.

To solve this, we rewrote the Verifier into a true Autonomous Agent using the ReAct (Reasoning & Acting) pattern. We armed the LLM with Python tools to search_web and scrape_url. Instead of a single-shot guess, the Agent now thinks: "I need the auth method. Let me search Google for 'App Developer API Auth'. Now let me click result #2. Ah, this page says API keys are used. I will stop." By allowing the agent to dynamically navigate the docs up to 5 times per app, accuracy skyrocketed past 70%.

Honest Hits & Misses:

HIT (Fathom): The Agent successfully read the Quickstart guide and extracted "API Key" as the primary auth. The Verifier cross-checked the text: "Generate an API key and make your first call in minutes." and validated the claim as True.
MISS (Linear): The Researcher claimed Linear's Buildability was "Ready". The Verifier read the documentation and found absolutely no mention of the word "Buildability" or its status. The Verifier correctly marked the claim as False / Unsupported because the LLM made a logical leap rather than strictly extracting data.
MISS (YouTube Transcript API): The Researcher claimed the Primary Auth was an "API Key". The Verifier read the PyPI documentation, found no mention of authentication, saw code examples initiating the client with no credentials, and accurately debunked the hallucination.

The Path to 99%

While the Agentic Loop fixed context starvation, the remaining ~30% gap is caused by modern web architecture (React SPAs, cookie walls, and JS-rendered API Explorers) blocking standard HTTP GET requests. The final evolution to reach 99% accuracy requires swapping our HTTP scraper with a Browser-Use / Playwright Agent that can physically execute JavaScript and click through consent banners just like a human developer.
